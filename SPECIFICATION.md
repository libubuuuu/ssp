# AI 创意平台 - 技术规格说明书

## 项目概述

全功能 AI 创意平台，包含：图片生成、视频生成、数字人三大核心模块，支持多窗口同步作业。

---

```

└─────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────┐
```

## 二、第一大功能：图片生成

### 2.1 风格化/广告级图片

| 项目 | 说明 |
|------|------|
| **模型** | SDXL / Flux + ControlNet + LoRA |
| **输入** | 用户描述、参考风格、尺寸、色彩倾向 |
| **输出** | 高清广告级图片 |
| **后处理** | 色彩增强、锐化、可选水印 |

**Prompt 模板示例**：
```
masterpiece, best quality, commercial advertising, professional lighting,
{用户描述}, {色彩搭配}, cinematic composition, 8k, ultra detailed
```

### 2.2 写实/可控图片

| 项目 | 说明 |
|------|------|
| **模型** | DALL·E 3 / Imagen 3 或 SD + GPT-4V 理解层 |
| **输入** | 自然语言描述，支持多轮细化 |
| **输出** | 严格遵循用户意图的图片 |
| **特点** | 不乱改、可局部编辑 (inpainting) |

## 三、第二大功能：视频生成

### 3.1 视频链接改造

**流程设计**：

```
1. 用户输入视频链接
   ↓
2. 系统下载/解析视频 (FFmpeg)
   ↓
3. 弹窗询问：
   - 是否需要替换人物/背景？ [全部/仅人物/仅背景/不需要]
   - 上传对应图片（若需要）
   ↓
4. 弹窗询问：
   - 产品与原视频一致吗？ [是/否]
   - 若否 → 上传产品图片
   ↓
5. 提取原视频：分镜、时长、景别、每镜提示词
   ↓
6. 展示：原提示词 vs 优化后提示词
   - 用户可选其一 / 在基础上修改
   ↓
7. 每次修改 → 记录到后台 → 同步飞书
   ↓
8. 生成新视频
```

**数据结构**：

```json
{
  "video_url": "https://...",
  "replace_mode": "character|background|both|none",
  "uploaded_images": {
    "character": "url",
    "background": "url",
    "product": "url"
  },
  "product_same": true,
  "shots": [
    {
      "index": 1,
      "duration_sec": 3.5,
      "scene_type": "close-up",
      "original_prompt": "...",
      "optimized_prompt": "...",
      "user_modified_prompt": "...",
      "edit_history": []
    }
  ]
}
```

**飞书同步**：每次 `user_modified_prompt` 变更时，POST 到飞书 Webhook，写入多维表格。

### 3.2 文生视频

- 直接调用 Runway / Pika / Kling API
- 输入：文本描述、时长、尺寸
- 输出：视频 URL

### 3.3 图生视频 + 分镜工作流

**流程设计**：

```
镜头1工作流:
  上传: 首帧图 (或 首帧+尾帧)
  填写: 脚本、分镜描述、时长
  ↓
镜头2工作流:
  自动继承上一镜头末帧 / 用户上传新首帧
  填写: 脚本、分镜描述、时长
  ↓
... 可继续添加镜头 ...
  ↓
生成: 保持人物一致性 + 镜头间连贯性
```

**技术要点**：
- **人物一致性**：IP-Adapter / InstantID 固定人物特征
- **连贯性**：帧插值、平滑转场、尽量一镜到底观感

## 四、第三大功能：数字人 AI

| 项目 | 说明 |
|------|------|
| **输入** | 人物图片 + 文本/音频脚本 |
| **输出** | 口型精准、无多余动作的数字人视频 |
| **模型** | SadTalker / D-ID / HeyGen 等 |

---

## 五、多窗口同步

**实现方式**：
- 任务 ID 全局唯一，多窗口共享
- WebSocket 推送任务状态变更
- 前端：同一任务在不同标签页显示相同进度
- 任务队列限流，避免并发过高

---

## 六、技术选型建议

| 层级 | 推荐 |
|------|------|
| 前端 | React / Next.js |
| 后端 | Node.js (Nest.js) 或 Python (FastAPI) |
| 任务队列 | Redis + Bull / Celery |
| 存储 | 对象存储 (阿里云 OSS / AWS S3) |
| 数据库 | PostgreSQL |
| 飞书 | 飞书开放平台 Webhook + 多维表格 |

---

## 七、开发阶段建议

1. **Phase 1**：图片生成（风格化 + 写实） MVP
2. **Phase 2**：文生视频 + 数字人
3. **Phase 3**：视频链接改造（最复杂）
4. **Phase 4**：图生视频工作流
5. **Phase 5**：多窗口同步 + 飞书集成优化
